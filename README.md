# РК1 по КС: 4 отдельные части

В проекте реализованы 4 алгоритма из задания:

1. `CRC-32` — клиент-сервер, обнаружение ошибки.
2. `Код Хэмминга` — клиент-сервер, обнаружение и исправление одиночной ошибки.
3. `Шеннон-Фано` — сжатие файла.
4. `Хаффман` — сжатие файла.

Главная идея текущей версии проекта: **каждая часть запускается отдельно в терминале** и показывает все ключевые этапы, которые нужны для сдачи.

## Самое важное: 4 команды для сдачи

### 1. CRC-32

```bash
python crc_demo.py --message "Привет, КС!" --error-bit 5
```

Что показывает терминал:

- исходное сообщение;
- перевод в байты и биты;
- вычисленную `CRC-32`;
- сформированный пакет `message + crc`;
- искусственно испорченный бит;
- передачу пакета на сервер;
- обнаружение ошибки на сервере;
- восстановление сообщения через повторную передачу правильного пакета.

Важно:

- `--error-bit` выбирается вручную тобой;
- индекс бита задаётся в формате `0-based`.

### 2. Код Хэмминга

```bash
python hamming_demo.py --message "KS" --error-bit 7
```

Что показывает терминал:

- исходное сообщение;
- бинарный вид сообщения;
- позиции служебных битов `1, 2, 4, 8, ...`;
- расстановку информационных и служебных битов;
- вычисление каждого служебного бита по правилу:
  `взять N бит, пропустить N бит`;
- готовое кодовое слово;
- искусственно испорченный бит;
- передачу на сервер;
- вычисление синдрома на сервере;
- позицию ошибочного бита;
- исправление и восстановление исходного сообщения.

Важно:

- `--error-bit` тоже выбирается вручную;
- для самой наглядной демонстрации лучше короткое сообщение, например `"KS"`.

### 3. Шеннон-Фано

```bash
python shannon_fano_demo.py
```

Или со своим файлом:

```bash
python shannon_fano_demo.py path/to/file.txt
```

Что показывает терминал:

- исходный файл и его размер;
- превью содержимого;
- частоты символов;
- построенную кодовую таблицу;
- длину битового потока;
- размер сжатого результата;
- восстановление файла;
- проверку, что распакованный файл совпадает с исходным.

### 4. Хаффман

```bash
python huffman_demo.py
```

Или со своим файлом:

```bash
python huffman_demo.py path/to/file.txt
```

Что показывает терминал:

- исходный файл и его размер;
- превью содержимого;
- частоты символов;
- кодовую таблицу Хаффмана;
- длину битового потока;
- размер результата;
- восстановление файла;
- проверку совпадения с исходником.

## Что именно соответствует заданию

### CRC-32

Реализовано как клиент-серверное приложение:

- сообщение берётся в текстовом виде;
- переводится в бинарный вид;
- кодируется с добавлением `CRC-32`;
- передаётся с ошибкой;
- сервер показывает, что ошибка есть;
- затем сообщение восстанавливается через повторную передачу правильного пакета.

Важно для защиты:

- `CRC-32` **обнаруживает**, но не исправляет ошибку сам.

### Код Хэмминга

Реализовано как клиент-серверное приложение:

- сообщение переводится в бинарный вид;
- служебные биты ставятся на позиции степеней двойки;
- каждый служебный бит считается не по таблице, а по правилу:
  `каждые N бит учитываем, следующие N пропускаем`;
- сообщение передаётся с ошибкой;
- сервер показывает синдром;
- сервер исправляет один бит и восстанавливает сообщение.

### Шеннон-Фано

Реализовано без клиент-сервера:

- берётся файл;
- строятся частоты;
- строится кодовая таблица;
- файл сжимается;
- файл распаковывается обратно;
- сравнивается результат.

### Хаффман

Реализовано без клиент-сервера:

- берётся файл;
- строится дерево и кодовая таблица;
- файл сжимается;
- файл распаковывается обратно;
- результат сравнивается с исходным.

## Краткие замечания для защиты

- `CRC-32` показывает обнаружение ошибки по контрольной сумме.
- `Хэмминг` показывает исправление одиночной ошибки по синдрому.
- `Шеннон-Фано` и `Хаффман` показывают сжатие без потерь.
- На маленьких файлах итоговый `.ksc` может оказаться больше исходного файла из-за заголовка с кодовой таблицей.
